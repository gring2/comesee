Key Feature: 오프라인 친화 로컬 네트워크로 친구들과 같은 사진을 볼 수 있는 앱(바이너리 전송 지원).

feature list
1. iPhone 사진첩(Photo Library)의 **사진만** 보는 뷰어. 비디오·Live Photo는 자동 필터링하거나 썸네일만 노출.
 1-1 호스트가 스와이프하여 공유할 사진을 선택 가능.
 1-2 권한: iOS Photo Library read-only(최소 권한). 비디오 접근을 받더라도 UI/전송 모두 사진만 노출/허용.

2. 로컬 근거리 통신으로 근처(기본 약 5m, 네트워크 품질에 따라 확장) 사용자와 동기화된 사진 보기.
 2-1 네트워크: BLE 광고로 호스트 발견, 실제 전송은 Wi‑Fi Direct(Local P2P) 또는 동일 LAN 시 P2P TCP 소켓. 바이너리 전송, TLS/DTLS 암호화.
 2-2 최대 동시 참여자 목표 6~8명(TBD). 호스트가 인원 제한 가능.
 2-3 모든 데이터는 기기 간 직접 전송, 외부 서버 미사용.

3. 프라이버시 지연(privacy buffer)으로 의도치 않은 사진 공유 방지.
 3-1 기본 지연 시간 2초(제품 설정에서 조정 가능). 지연 타이머 안에 호스트가 “공유 취소” 시 전송 없음.
 3-2 지연 중 게스트 화면: 이전 사진 유지 또는 로딩 플레이스홀더(UX 결정). 취소 시 이전 사진으로 복귀.
 3-3 호스트가 빠르게 스크럽할 때는 마지막 멈춘 사진만 전송(중간 사진 드롭)해 과도한 전송을 억제.

4. 세션/참여 플로우
 4-1 호스트: 앱 실행 → “공유 시작” → BLE 광고 + Wi‑Fi Direct 세션 생성.
 4-2 게스트: 앱 실행 상태에서 호스트 발견 푸시/배너 수신 → “참여” 선택 시 입장. 거절 시 종료.
 4-3 세션 종료: 호스트 종료 또는 모든 게스트 퇴장 시 종료. 호스트가 특정 게스트 강제 퇴장/차단(TBD) 가능.
 4-4 중간 합류: 늦게 들어온 게스트는 호스트의 현재 사진부터 수신(히스토리 미전송).

5. 성능/제한
 - 목표 체감 지연: 호스트 스와이프부터 게스트 렌더까지 300~500ms(프라이버시 지연 제외).
 - 사진 해상도: 화면 해상도 맞춤 리사이즈·압축 후 전송해 배터리/대역폭 최소화. 원본 저장 없음.
 - 캐시: 세션 동안 임시 캐시, 세션 종료 시 자동 삭제.
